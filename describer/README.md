# SillyTavern Immersion E-Stim Analyzer

A powerful, two-step Python pipeline designed to analyze audio-based E-Stim (Electrical Stimulation) files and generate highly detailed, context-aware sensation descriptions. Using audio analysis alongside a multimodal LLM (via OpenRouter), this tool creates a perfectly formatted JSON file for the SillyTavern E-Stim Extension, bridging the gap between digital roleplay and physical bio-feedback.

## 🌟 Features

* **E-Stim Optimized Visuals:** Automatically converts `.mp3` audio files into high-resolution spectrograms and waveforms. It is specifically calibrated for E-Stim signals (cutting off irrelevant frequencies above 1.5 kHz and ignoring inaudible noise floors).
* **Stereo & Mono Support:** Dynamically handles both stereo and mono files, separating channels (CH1/CH2) to visualize carrier frequencies and amplitude envelopes independently.
* **Context-Aware LLM Analysis:** Feeds the generated spectrograms along with optional story scripts (context) to a multimodal LLM (default: `google/gemini-3.1-pro-preview`).
* **SillyTavern Ready Output:** Outputs a complete `estim_sensations.json` file, categorizing files by intensity, temporal progression, and contextual intent, utilizing `{{ESTIM_CH1}}` and `{{ESTIM_CH2}}` macros for dynamic roleplay integration.
* **Cost Transparent:** Calculates and displays API token usage before and after sending data to OpenRouter.

## 🛠️ Prerequisites & Installation

It is highly recommended to run this project inside a Python Virtual Environment (`venv`) to avoid dependency conflicts.

**1. Clone the repository and navigate into it:**

```bash
git clone https://github.com/ark2398/st-estim-extension.git
cd st-estim-extension

```

**2. Create and activate a virtual environment:**

* **Windows:**
```powershell
python -m venv env
.\env\Scripts\activate

```


*(Note: If PowerShell blocks the script, run `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` first).*
* **macOS / Linux:**
```bash
python3 -m venv env
source env/bin/activate

```



**3. Install required packages:**

```bash
pip install librosa matplotlib numpy requests python-dotenv

```

**4. Set up your OpenRouter API Key:**
Create a file named `.env` in the root directory of the project and add your OpenRouter API key:

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here

```

## 🚀 Usage Pipeline

The workflow consists of two scripts that must be run in order.

### Step 1: Generate Spectrograms (`describe-spectrum.py`)

This script analyzes your E-Stim audio files and generates technical, high-resolution plots.

1. Place your `.mp3` files in the `./audio` folder (the script will create this folder on the first run if it doesn't exist).
2. Run the analyzer:
```bash
python describe-spectrum.py

```


3. The generated PNG images will be saved in the `./spectrum` folder.

### Step 2: Generate Sensation Descriptions (`describe-llm.py`)

This script takes the images from Step 1 and asks an LLM to interpret the physical sensations they will cause.

1. **(Optional but Recommended):** Place any relevant story scripts, code snippets, or dialogue text (e.g., `.txt`, `.html`, `.js`) into the `./context` folder. The LLM will read these files to understand the *narrative intent* behind the audio files (e.g., teasing vs. harsh punishment).
2. Run the generator:
```bash
python describe-llm.py

```


3. Review the terminal output. The script will tell you how many images and context files it found. Type `y` or `yes` to confirm and send the data to OpenRouter.
4. Once completed, you will see your API token usage, and the final output will be saved as `estim_sensations.json`.

## 📂 Folder Structure

* `describe-spectrum.py`: The audio analysis script.
* `describe-llm.py`: The LLM generation script.
* `.env`: Your hidden environment variables (do not commit this to Git).
* `/audio/`: Place your input `.mp3` files here.
* `/spectrum/`: Auto-generated spectrogram/waveform images are saved here.
* `/context/`: Drop text/code files here to give the LLM narrative context for the audio files.
* `estim_sensations.json`: The final output file generated by the LLM.
